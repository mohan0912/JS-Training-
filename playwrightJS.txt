1. INTRODUCTION TO PLAYWRIGHT
================================================================================

Playwright is a modern end-to-end testing framework developed by Microsoft.

--------------------------------------------------------------------------------
1.1 Key Features
--------------------------------------------------------------------------------

  • Cross-browser: Chromium, Firefox, and WebKit (Safari)
  • Cross-platform: Windows, macOS, Linux
  • Cross-language: JavaScript, TypeScript, Python, Java, .NET
  • Auto-waiting: Automatically waits for elements to be actionable
  • Built-in assertions: expect() with auto-retry
  • Parallel execution: Fast test runs out of the box
  • Tracing: Time-travel debugging with screenshots and DOM snapshots
  • API testing: Built-in support for REST API testing
  • Mobile emulation: Test responsive designs and mobile viewports

--------------------------------------------------------------------------------
1.2 Why Playwright Over Other Tools
--------------------------------------------------------------------------------

  • Faster than Selenium (direct browser protocol communication)
  • More reliable than Cypress (handles multiple tabs, iframes easily)
  • Modern architecture with auto-waiting
  • Excellent debugging tools (Inspector, Trace Viewer)
  • Strong TypeScript support
  • Active development and Microsoft backing

2. INSTALLATION AND SETUP
================================================================================

--------------------------------------------------------------------------------
2.1 Prerequisites
--------------------------------------------------------------------------------

  • Node.js 16 or higher (LTS recommended)
  • npm or yarn package manager
  • VS Code (recommended IDE)

    # Check Node.js version
    node --version

--------------------------------------------------------------------------------
2.2 Create New Playwright Project
--------------------------------------------------------------------------------

    # Initialize new project with prompts
    npm init playwright@latest

    # This will:
    # - Create playwright.config.js/ts
    # - Create tests folder with example test
    # - Create .github/workflows for CI
    # - Install browsers

--------------------------------------------------------------------------------
2.3 Add to Existing Project
--------------------------------------------------------------------------------

    # Install Playwright
    npm install -D @playwright/test

    # Install browsers
    npx playwright install

    # Install specific browser only
    npx playwright install chromium
    npx playwright install firefox
    npx playwright install webkit

--------------------------------------------------------------------------------
2.4 Install Browser Dependencies (Linux)
--------------------------------------------------------------------------------

    # Install system dependencies for browsers
    npx playwright install-deps

    # Install deps for specific browser
    npx playwright install-deps chromium

--------------------------------------------------------------------------------
2.5 VS Code Extension
--------------------------------------------------------------------------------

Install "Playwright Test for VS Code" extension for:
  • Run tests from sidebar
  • Debug tests with breakpoints
  • Record new tests
  • Pick locators
  • View test reports

4. CONFIGURATION (playwright.config.js)
================================================================================

--------------------------------------------------------------------------------
4.1 Basic Configuration
--------------------------------------------------------------------------------

    // playwright.config.js
    import { defineConfig, devices } from '@playwright/test';

    export default defineConfig({
        // Directory containing test files
        testDir: './tests',

        // Test file pattern
        testMatch: '**/*.spec.js',

        // Run tests in parallel
        fullyParallel: true,

        // Fail the build on CI if test.only is left in code
        forbidOnly: !!process.env.CI,

        // Retry failed tests
        retries: process.env.CI ? 2 : 0,

        // Number of parallel workers
        workers: process.env.CI ? 1 : undefined,

        // Reporter configuration
        reporter: 'html',

        // Shared settings for all projects
        use: {
            // Base URL for page.goto('/')
            baseURL: 'https://example.com',

            // Collect trace on failure
            trace: 'on-first-retry',

            // Take screenshot on failure
            screenshot: 'only-on-failure',

            // Record video on failure
            video: 'retain-on-failure',
        },

        // Browser projects
        projects: [
            {
                name: 'chromium',
                use: { ...devices['Desktop Chrome'] },
            },
            {
                name: 'firefox',
                use: { ...devices['Desktop Firefox'] },
            },
            {
                name: 'webkit',
                use: { ...devices['Desktop Safari'] },
            },
        ],
    });

5. WRITING YOUR FIRST TEST
================================================================================

--------------------------------------------------------------------------------
5.1 Basic Test Structure
--------------------------------------------------------------------------------

    // tests/example.spec.js
    import { test, expect } from '@playwright/test';

    test('has title', async ({ page }) => {
        // Navigate to page
        await page.goto('https://playwright.dev/');

        // Assert title contains text
        await expect(page).toHaveTitle(/Playwright/);
    });

    test('get started link', async ({ page }) => {
        await page.goto('https://playwright.dev/');

        // Click the link
        await page.getByRole('link', { name: 'Get started' }).click();

        // Assert URL changed
        await expect(page).toHaveURL(/.*intro/);
    });

--------------------------------------------------------------------------------
5.2 Test with describe Block
--------------------------------------------------------------------------------

    import { test, expect } from '@playwright/test';

    test.describe('Login Feature', () => {
        test.beforeEach(async ({ page }) => {
            await page.goto('/login');
        });

        test('should display login form', async ({ page }) => {
            await expect(page.getByLabel('Email')).toBeVisible();
            await expect(page.getByLabel('Password')).toBeVisible();
            await expect(page.getByRole('button', { name: 'Login' })).toBeVisible();
        });

        test('should login with valid credentials', async ({ page }) => {
            await page.getByLabel('Email').fill('user@example.com');
            await page.getByLabel('Password').fill('password123');
            await page.getByRole('button', { name: 'Login' }).click();

            await expect(page).toHaveURL('/dashboard');
        });

        test('should show error for invalid credentials', async ({ page }) => {
            await page.getByLabel('Email').fill('invalid@example.com');
            await page.getByLabel('Password').fill('wrongpassword');
            await page.getByRole('button', { name: 'Login' }).click();

            await expect(page.getByText('Invalid credentials')).toBeVisible();
        });
    });

--------------------------------------------------------------------------------

6. LOCATOR STRATEGIES (CRITICAL)
================================================================================

Locators are the foundation of Playwright tests. Master these strategies!

--------------------------------------------------------------------------------
6.1 Recommended: Role-Based Locators (Best Practice)
--------------------------------------------------------------------------------
Use semantic roles for accessibility and reliability.

    // Buttons
    page.getByRole('button', { name: 'Submit' })
    page.getByRole('button', { name: /submit/i })  // Regex, case-insensitive

    // Links
    page.getByRole('link', { name: 'Home' })

    // Text inputs
    page.getByRole('textbox', { name: 'Email' })
    page.getByRole('spinbutton', { name: 'Quantity' })  // Number input

    // Checkboxes and radio buttons
    page.getByRole('checkbox', { name: 'Accept terms' })
    page.getByRole('radio', { name: 'Credit Card' })

    // Dropdowns
    page.getByRole('combobox', { name: 'Country' })

    // Headings
    page.getByRole('heading', { name: 'Welcome' })
    page.getByRole('heading', { level: 1 })  // h1

    // Navigation, lists, etc.
    page.getByRole('navigation')
    page.getByRole('list')
    page.getByRole('listitem')
    page.getByRole('table')
    page.getByRole('row')
    page.getByRole('cell', { name: 'Price' })

    // With exact match
    page.getByRole('button', { name: 'Submit', exact: true })

--------------------------------------------------------------------------------
6.2 Label and Placeholder Locators
--------------------------------------------------------------------------------
Find inputs by their associated label or placeholder text.

    // By label (recommended for form fields)
    page.getByLabel('Email')
    page.getByLabel('Password')
    page.getByLabel(/email/i)  // Regex

    // By placeholder
    page.getByPlaceholder('Enter your email')
    page.getByPlaceholder(/email/i)

--------------------------------------------------------------------------------
6.3 Text Locators
--------------------------------------------------------------------------------
Find elements by their text content.

    // Exact text
    page.getByText('Hello World')

    // Partial text
    page.getByText('Hello')

    // Regex
    page.getByText(/hello/i)

    // Exact match only
    page.getByText('Hello', { exact: true })

--------------------------------------------------------------------------------
6.4 Test ID Locators (Highly Recommended)
--------------------------------------------------------------------------------
Best for elements without clear text/role. Add data-testid to your HTML.

    // HTML: <button data-testid="submit-btn">Submit</button>
    page.getByTestId('submit-btn')

    // Configure custom test ID attribute
    // In playwright.config.js:
    use: {
        testIdAttribute: 'data-test-id'
    }

--------------------------------------------------------------------------------
6.5 Alt Text Locator
--------------------------------------------------------------------------------
Find images by alt text.

    page.getByAltText('Company Logo')
    page.getByAltText(/logo/i)

--------------------------------------------------------------------------------
6.6 Title Locator
--------------------------------------------------------------------------------
Find elements by title attribute.

    page.getByTitle('Close dialog')
    page.getByTitle(/close/i)

--------------------------------------------------------------------------------
6.7 CSS Selectors
--------------------------------------------------------------------------------
Traditional CSS selectors when other methods don't work.

    // By ID
    page.locator('#submit-button')

    // By class
    page.locator('.btn-primary')
    page.locator('.btn.large.primary')  // Multiple classes

    // By tag
    page.locator('button')

    // By attribute
    page.locator('[data-id="123"]')
    page.locator('[type="submit"]')
    page.locator('input[type="email"]')

    // Partial attribute match
    page.locator('[class*="btn"]')      // Contains
    page.locator('[class^="btn"]')      // Starts with
    page.locator('[class$="primary"]')  // Ends with

    // Combinators
    page.locator('form button')         // Descendant
    page.locator('form > button')       // Direct child
    page.locator('label + input')       // Adjacent sibling
    page.locator('h1 ~ p')              // General sibling

    // Pseudo-selectors
    page.locator('li:first-child')
    page.locator('li:last-child')
    page.locator('li:nth-child(2)')
    page.locator('tr:nth-child(odd)')
    page.locator('input:not([disabled])')

--------------------------------------------------------------------------------
6.8 XPath Selectors
--------------------------------------------------------------------------------
Use when CSS selectors are insufficient.

    // Basic XPath
    page.locator('//button[@type="submit"]')

    // By text
    page.locator('//button[text()="Submit"]')
    page.locator('//button[contains(text(), "Submit")]')

    // By attribute
    page.locator('//*[@data-testid="submit"]')

    // Parent/ancestor
    page.locator('//span[@class="error"]/parent::div')
    page.locator('//input[@id="email"]/ancestor::form')

    // Following/preceding sibling
    page.locator('//label[text()="Email"]/following-sibling::input')

--------------------------------------------------------------------------------
6.9 Chaining and Filtering Locators
--------------------------------------------------------------------------------
Combine locators for precise targeting.

    // Chain locators (find within)
    page.locator('form').locator('button')
    page.getByRole('table').getByRole('row').first()

    // Filter by text
    page.locator('button').filter({ hasText: 'Submit' })

    // Filter by another locator
    page.locator('li').filter({ has: page.getByRole('button') })

    // Filter by NOT having
    page.locator('li').filter({ hasNot: page.locator('.disabled') })

    // Combining with nth
    page.locator('li').nth(2)       // Third item (0-indexed)
    page.locator('li').first()      // First item
    page.locator('li').last()       // Last item

--------------------------------------------------------------------------------
6.10 Locator List Operations
--------------------------------------------------------------------------------
Work with multiple matching elements.

    // Count elements
    const count = await page.locator('li').count();

    // Get all elements
    const items = page.locator('li');
    for (const item of await items.all()) {
        console.log(await item.textContent());
    }

    // Get all text contents
    const texts = await page.locator('li').allTextContents();

    // Get all inner texts
    const innerTexts = await page.locator('li').allInnerTexts();

--------------------------------------------------------------------------------
6.11 Frame Locators
--------------------------------------------------------------------------------
Work with iframes.

    // Locate frame and then element inside
    page.frameLocator('#iframe-id').getByRole('button', { name: 'Submit' })

    // Chain frame locators
    page.frameLocator('#outer').frameLocator('#inner').locator('button')

--------------------------------------------------------------------------------
6.12 Locator Best Practices
--------------------------------------------------------------------------------

  PREFER (most to least):
  1. getByRole() - Accessible, reflects user experience
  2. getByLabel() - For form inputs
  3. getByPlaceholder() - For inputs without labels
  4. getByText() - For static text elements
  5. getByTestId() - When semantic locators don't work
  6. CSS selectors - When above don't work
  7. XPath - Last resort

  AVOID:
  - Brittle selectors: div.class1.class2 > span:nth-child(3)
  - Auto-generated IDs: #react-select-123-option-0
  - Positional only: nth-child without context

7. ACTIONS AND INTERACTIONS
================================================================================

--------------------------------------------------------------------------------
7.1 Click Actions
--------------------------------------------------------------------------------

    // Basic click
    await page.getByRole('button', { name: 'Submit' }).click();

    // Double click
    await page.locator('.item').dblclick();

    // Right click (context menu)
    await page.locator('.item').click({ button: 'right' });

    // Shift+click
    await page.locator('.item').click({ modifiers: ['Shift'] });

    // Ctrl+click (Cmd on Mac)
    await page.locator('.item').click({ modifiers: ['Control'] });

    // Click at specific position
    await page.locator('.canvas').click({ position: { x: 100, y: 200 } });

    // Force click (bypass actionability checks)
    await page.locator('.hidden-btn').click({ force: true });

    // Click with timeout
    await page.locator('button').click({ timeout: 5000 });

    // Trial click (don't actually click, just check if actionable)
    await page.locator('button').click({ trial: true });

--------------------------------------------------------------------------------
7.2 Type and Fill Actions
--------------------------------------------------------------------------------

    // Fill (clears and types) - RECOMMENDED
    await page.getByLabel('Email').fill('user@example.com');

    // Type (types character by character)
    await page.getByLabel('Search').type('playwright');

    // Type with delay between keystrokes
    await page.locator('input').type('hello', { delay: 100 });

    // Press and type (for special keys)
    await page.getByLabel('Email').pressSequentially('user@test.com');

    // Clear input
    await page.getByLabel('Email').clear();

--------------------------------------------------------------------------------
7.3 Keyboard Actions
--------------------------------------------------------------------------------

    // Press single key
    await page.keyboard.press('Enter');
    await page.keyboard.press('Tab');
    await page.keyboard.press('Escape');
    await page.keyboard.press('Backspace');

    // Key combinations
    await page.keyboard.press('Control+A');      // Select all
    await page.keyboard.press('Control+C');      // Copy
    await page.keyboard.press('Control+V');      // Paste
    await page.keyboard.press('Control+Z');      // Undo
    await page.keyboard.press('Shift+Tab');      // Reverse tab

    // On specific element
    await page.getByLabel('Name').press('Enter');

    // Hold and release
    await page.keyboard.down('Shift');
    await page.keyboard.press('ArrowRight');
    await page.keyboard.up('Shift');

    // Type text via keyboard
    await page.keyboard.insertText('Hello World');

--------------------------------------------------------------------------------
7.4 Checkbox and Radio Button
--------------------------------------------------------------------------------

    // Check checkbox
    await page.getByRole('checkbox', { name: 'Accept terms' }).check();

    // Uncheck checkbox
    await page.getByRole('checkbox', { name: 'Newsletter' }).uncheck();

    // Set checkbox state (check or uncheck based on boolean)
    await page.getByRole('checkbox').setChecked(true);
    await page.getByRole('checkbox').setChecked(false);

    // Check radio button
    await page.getByRole('radio', { name: 'Credit Card' }).check();

    // Assert checkbox state
    await expect(page.getByRole('checkbox')).toBeChecked();
    await expect(page.getByRole('checkbox')).not.toBeChecked();

--------------------------------------------------------------------------------
7.5 Select Dropdown
--------------------------------------------------------------------------------

    // Select by value
    await page.getByLabel('Country').selectOption('us');

    // Select by label text
    await page.getByLabel('Country').selectOption({ label: 'United States' });

    // Select by index
    await page.getByLabel('Country').selectOption({ index: 2 });

    // Select multiple options
    await page.getByLabel('Colors').selectOption(['red', 'green', 'blue']);

    // Clear selection
    await page.getByLabel('Country').selectOption([]);

--------------------------------------------------------------------------------
7.6 Hover and Focus
--------------------------------------------------------------------------------

    // Hover over element
    await page.getByText('Menu').hover();

    // Focus element
    await page.getByLabel('Email').focus();

    // Blur (unfocus) element
    await page.getByLabel('Email').blur();

--------------------------------------------------------------------------------
7.7 Drag and Drop
--------------------------------------------------------------------------------

    // Drag and drop to another element
    await page.locator('#item').dragTo(page.locator('#target'));

    // Manual drag and drop
    await page.locator('#item').hover();
    await page.mouse.down();
    await page.locator('#target').hover();
    await page.mouse.up();

--------------------------------------------------------------------------------
7.8 Scrolling
--------------------------------------------------------------------------------

    // Scroll element into view
    await page.locator('#footer').scrollIntoViewIfNeeded();

    // Scroll by amount
    await page.mouse.wheel(0, 500);  // Scroll down 500px

    // Scroll within element
    await page.locator('.scroll-container').evaluate(el => {
        el.scrollTop = 500;
    });

--------------------------------------------------------------------------------
7.9 Mouse Actions
--------------------------------------------------------------------------------

    // Move mouse
    await page.mouse.move(100, 200);

    // Mouse down/up
    await page.mouse.down();
    await page.mouse.up();

    // Click at coordinates
    await page.mouse.click(100, 200);

    // Double click
    await page.mouse.dblclick(100, 200);

--------------------------------------------------------------------------------
7.10 Touch Actions (Mobile)
--------------------------------------------------------------------------------

    // Tap
    await page.locator('button').tap();

    // Touch screen
    await page.touchscreen.tap(100, 200);

8. ASSERTIONS AND EXPECTATIONS
================================================================================

Playwright's expect() function has auto-retry built in.

--------------------------------------------------------------------------------
8.1 Page Assertions
--------------------------------------------------------------------------------

    // URL assertions
    await expect(page).toHaveURL('https://example.com');
    await expect(page).toHaveURL(/.*dashboard/);
    await expect(page).toHaveURL(/\/login$/);

    // Title assertions
    await expect(page).toHaveTitle('Dashboard');
    await expect(page).toHaveTitle(/Dashboard/);

--------------------------------------------------------------------------------
8.2 Locator Visibility Assertions
--------------------------------------------------------------------------------

    const button = page.getByRole('button', { name: 'Submit' });

    // Visibility
    await expect(button).toBeVisible();
    await expect(button).toBeHidden();
    await expect(button).not.toBeVisible();

    // Attached to DOM (may be hidden)
    await expect(button).toBeAttached();

--------------------------------------------------------------------------------
8.3 Locator State Assertions
--------------------------------------------------------------------------------

    const input = page.getByLabel('Email');

    // Enabled/Disabled
    await expect(input).toBeEnabled();
    await expect(input).toBeDisabled();

    // Editable
    await expect(input).toBeEditable();

    // Focused
    await expect(input).toBeFocused();

    // Checked (checkbox/radio)
    await expect(page.getByRole('checkbox')).toBeChecked();
    await expect(page.getByRole('checkbox')).not.toBeChecked();

--------------------------------------------------------------------------------
8.4 Locator Text Assertions
--------------------------------------------------------------------------------

    const heading = page.getByRole('heading');
    const message = page.locator('.message');

    // Exact text
    await expect(heading).toHaveText('Welcome');

    // Partial text
    await expect(message).toContainText('Success');

    // Regex
    await expect(heading).toHaveText(/welcome/i);

    // Multiple elements
    await expect(page.locator('li')).toHaveText(['Item 1', 'Item 2', 'Item 3']);

--------------------------------------------------------------------------------
8.5 Locator Value Assertions
--------------------------------------------------------------------------------

    const input = page.getByLabel('Email');

    // Input value
    await expect(input).toHaveValue('user@example.com');
    await expect(input).toHaveValue(/example\.com$/);

    // Empty value
    await expect(input).toBeEmpty();
    await expect(input).not.toBeEmpty();

    // Multiple values (multi-select)
    await expect(page.locator('select')).toHaveValues(['red', 'blue']);

--------------------------------------------------------------------------------
8.6 Locator Attribute Assertions
--------------------------------------------------------------------------------

    const link = page.getByRole('link', { name: 'Home' });

    // Has attribute
    await expect(link).toHaveAttribute('href', '/home');
    await expect(link).toHaveAttribute('href', /home/);

    // Has class
    await expect(link).toHaveClass('nav-link');

9. WAITING STRATEGIES
================================================================================

Playwright has auto-waiting, but sometimes explicit waits are needed.

--------------------------------------------------------------------------------
9.1 Auto-Waiting (Built-in)
--------------------------------------------------------------------------------
Most actions automatically wait for elements to be:
  • Attached to DOM
  • Visible
  • Stable (not animating)
  • Enabled
  • Receiving events (not obscured)

    // These automatically wait:
    await page.click('button');           // Waits for button to be clickable
    await page.fill('input', 'text');     // Waits for input to be editable
    await expect(locator).toBeVisible();  // Retries until visible or timeout

--------------------------------------------------------------------------------
9.2 Wait for Element State
--------------------------------------------------------------------------------

    const button = page.locator('button');

    // Wait for specific state
    await button.waitFor();                           // Default: visible
    await button.waitFor({ state: 'visible' });
    await button.waitFor({ state: 'hidden' });
    await button.waitFor({ state: 'attached' });
    await button.waitFor({ state: 'detached' });

    // With timeout
    await button.waitFor({ state: 'visible', timeout: 10000 });

--------------------------------------------------------------------------------
9.3 Wait for Page Events
--------------------------------------------------------------------------------

    // Wait for navigation
    await page.waitForURL('**/dashboard');
    await page.waitForURL(/.*dashboard.*/);
    await page.waitForURL(url => url.pathname === '/dashboard');

    // Wait for load state
    await page.waitForLoadState('load');              // Default
    await page.waitForLoadState('domcontentloaded');
    await page.waitForLoadState('networkidle');       // No network for 500ms

--------------------------------------------------------------------------------
9.4 Wait for Network
--------------------------------------------------------------------------------

    // Wait for specific request
    const requestPromise = page.waitForRequest('**/api/users');
    await page.click('button');
    const request = await requestPromise;

    // Wait for specific response
    const responsePromise = page.waitForResponse('**/api/users');
    await page.click('button');
    const response = await responsePromise;
    const data = await response.json();

    // Wait for response with condition
    const responsePromise = page.waitForResponse(
        response => response.url().includes('/api/users') && response.status() === 200
    );

--------------------------------------------------------------------------------
9.5 Wait for Function (Custom Condition)
--------------------------------------------------------------------------------

    // Wait for custom JavaScript condition
    await page.waitForFunction(() => {
        return document.querySelectorAll('.item').length > 5;
    });

    // With arguments
    await page.waitForFunction(
        (count) => document.querySelectorAll('.item').length > count,
        10  // argument
    );

    // With options
    await page.waitForFunction(
        () => window.dataLoaded === true,
        { timeout: 30000, polling: 1000 }
    );

--------------------------------------------------------------------------------
9.6 Wait with Promise.all (Parallel)
--------------------------------------------------------------------------------

    // Wait for navigation AND click simultaneously
    await Promise.all([
        page.waitForURL('**/dashboard'),
        page.click('button'),
    ]);

    // Wait for response AND action
    const [response] = await Promise.all([
        page.waitForResponse('**/api/submit'),
        page.click('#submit'),
    ]);

--------------------------------------------------------------------------------
9.7 Wait for Event
--------------------------------------------------------------------------------

    // Wait for download
    const downloadPromise = page.waitForEvent('download');
    await page.click('#download-btn');
    const download = await downloadPromise;

    // Wait for popup
    const popupPromise = page.waitForEvent('popup');
    await page.click('#open-popup');
    const popup = await popupPromise;

    // Wait for dialog
    page.on('dialog', dialog => dialog.accept());
    // or
    const dialogPromise = page.waitForEvent('dialog');
    await page.click('#show-alert');
    const dialog = await dialogPromise;

--------------------------------------------------------------------------------
9.8 Fixed Timeout (Avoid If Possible)
--------------------------------------------------------------------------------

    // Hard wait - NOT RECOMMENDED but sometimes necessary
    await page.waitForTimeout(2000);  // Wait 2 seconds

    // Better alternative: wait for specific condition
    await page.locator('.loaded').waitFor();

30.1 Retry on Failure
--------------------------------------------------------------------------------

    async function retryAction(action, maxAttempts = 3) {
        for (let attempt = 1; attempt <= maxAttempts; attempt++) {
            try {
                return await action();
            } catch (error) {
                if (attempt === maxAttempts) throw error;
                console.log(`Attempt ${attempt} failed, retrying...`);
            }
        }
    }

    // Usage
    await retryAction(() => page.click('#flaky-button'));

30.3 Handle Loading States

    async function waitForLoadingComplete(page) {
        // Wait for loader to appear then disappear
        const loader = page.locator('.loading-spinner');
        await loader.waitFor({ state: 'visible', timeout: 1000 }).catch(() => {});
        await loader.waitFor({ state: 'hidden' });
    }

30.5 Pagination Pattern
--------------------------------------------------------------------------------

    async function getAllItems(page) {
        const items = [];
        
        while (true) {
            const pageItems = await page.locator('.item').allTextContents();
            items.push(...pageItems);
            
            const nextButton = page.locator('.next-page');
            if (await nextButton.isDisabled()) break;
            
            await nextButton.click();
            await page.waitForLoadState('networkidle');
        }
        
        return items;
    }

--------------------------------------------------------------------------------
30.6 Table Data Extraction
--------------------------------------------------------------------------------

    async function getTableData(page, tableSelector) {
        const rows = page.locator(`${tableSelector} tbody tr`);
        const count = await rows.count();
        const data = [];
        
        for (let i = 0; i < count; i++) {
            const cells = rows.nth(i).locator('td');
            const rowData = await cells.allTextContents();
            data.push(rowData);
        }
        
        return data;
    }

--------------------------------------------------------------------------------
30.7 Form Fill Pattern
--------------------------------------------------------------------------------

    async function fillForm(page, formData) {
        for (const [field, value] of Object.entries(formData)) {
            const input = page.getByLabel(field);
            const tagName = await input.evaluate(el => el.tagName);
            
            if (tagName === 'SELECT') {
                await input.selectOption(value);
            } else if (await input.getAttribute('type') === 'checkbox') {
                if (value) await input.check();
                else await input.uncheck();
            } else {
                await input.fill(value);
            }
        }
    }

    // Usage
    await fillForm(page, {
        'Email': 'user@example.com',
        'Password': 'password123',
        'Remember me': true,
    });

30.8 Drag and Drop Pattern
--------------------------------------------------------------------------------

    async function dragAndDrop(page, sourceSelector, targetSelector) {
        const source = page.locator(sourceSelector);
        const target = page.locator(targetSelector);
        
        await source.dragTo(target);
    }

--------------------------------------------------------------------------------
30.9 Modal/Dialog Pattern
--------------------------------------------------------------------------------

    async function handleModal(page, action = 'confirm') {
        const modal = page.locator('.modal');
        await modal.waitFor({ state: 'visible' });
        
        if (action === 'confirm') {
            await modal.getByRole('button', { name: 'Confirm' }).click();
        } else {
            await modal.getByRole('button', { name: 'Cancel' }).click();
        }
        
        await modal.waitFor({ state: 'hidden' });
    }

--------------------------------------------------------------------------------
31.1 Element Not Found
--------------------------------------------------------------------------------

    // Problem: Element not found
    // Solutions:
    
    // 1. Check if element is in iframe
    const frame = page.frameLocator('iframe');
    await frame.locator('button').click();
    
    // 2. Wait for element explicitly
    await page.locator('button').waitFor();
    
    // 3. Check if element is attached but hidden
    await expect(page.locator('button')).toBeAttached();
    
    // 4. Use more specific locator
    await page.getByRole('button', { name: 'Submit' }).click();

--------------------------------------------------------------------------------
31.2 Timeout Errors
--------------------------------------------------------------------------------

    // Problem: Test timeout
    // Solutions:
    
    // 1. Increase timeout
    await page.click('button', { timeout: 30000 });
    
    // 2. Check network issues
    await page.waitForLoadState('networkidle');
    
    // 3. Debug with pause
    await page.pause();
    
    // 4. Use trace viewer
    // npx playwright test --trace on

--------------------------------------------------------------------------------
31.3 Flaky Tests
--------------------------------------------------------------------------------

    // Problem: Tests pass sometimes, fail sometimes
    // Solutions:
    
    // 1. Add retries
    // playwright.config.js: retries: 2
    
    // 2. Wait for stable state
    await page.locator('.element').waitFor({ state: 'visible' });
    
    // 3. Use web-first assertions
    await expect(page.locator('.element')).toBeVisible();
    
    // 4. Mock external dependencies
    await page.route('**/api/external', route => route.fulfill({
        status: 200,
        body: JSON.stringify({ data: 'mocked' }),
    }));

    // Problem: Tests fail after login
